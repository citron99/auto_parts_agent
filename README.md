# Auto Parts Agent (LangChain + OpenAI)

Учебный проект: базовый агент на LangChain, который помогает подбирать автозапчасти для интернет‑магазина по текстовому запросу пользователя.

Агент уточняет марку, модель и год автомобиля, вызывает инструмент поиска по каталогу и возвращает найденную запчасть с названием, артикулом и ценой. [web:2]

---

## Функциональность

- Принимает текстовый запрос пользователя (например: «нужны передние тормозные колодки на Toyota Corolla 2015»).
- При необходимости задаёт уточняющие вопросы: марку, модель и год машины.
- Использует инструмент `find_part_in_catalog` для поиска подходящей запчасти в локальном каталоге (`catalog.py`).
- Возвращает структурированный ответ:
  - марка, модель, год;
  - название детали;
  - артикул (SKU/OEM);
  - цена и валюта;
  - короткое пояснение/комментарий. [web:2]

## Установка и запуск

1. Клонируй / скачай проект в папку:

```bash
git clone https://github.com/<your-username>/auto_parts_agent.git
cd auto_parts_agent
```

2. Создай и активируй виртуальное окружение:

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
```

3. Установи зависимости:

```bash
pip install -r requirements.txt
```

4. Создай файл `.env` в корне проекта и вставь туда свой OpenAI ключ:

```env
OPENAI_API_KEY=sk-...твой_реальный_ключ
```

`ChatOpenAI` автоматически возьмёт ключ из переменной `OPENAI_API_KEY`. [web:30]

5. Запусти агента:

```bash
python main.py
```

## Как работает агент (компоненты LangChain)

### Цель агента

Цель — быть помощником интернет‑магазина автозапчастей, который:
- собирает необходимую информацию об автомобиле;
- подбирает запчасть по каталогу;
- выдаёт пользователю понятный ответ с конкретным товаром. [web:2]

### LLM

Используется `ChatOpenAI` (OpenAI GPT, например `gpt-4o-mini`).  
Модель:
- генерирует ответы пользователю;
- решает, когда нужно уточнить марку/модель/год;
- решает, когда вызвать инструмент поиска;
- формирует финальный ответ на основе результата инструмента. [web:2][web:30]

### Промпт

Промпт создаётся через `ChatPromptTemplate` и включает:
- системное сообщение с ролью агента (помощник магазина автозапчастей);
- правила поведения: сначала уточнять данные, затем искать;
- место для входного сообщения пользователя и scratchpad для рассуждений агента. [web:2]

### Инструменты (tools)

Единственный инструмент — функция `find_part_in_catalog`, помеченная `@tool`:
- принимает `brand`, `model`, `year`, `query`;
- ищет совпадения в `AUTO_PARTS_CATALOG` из `catalog.py`;
- возвращает(dict): найденная запчасть или сообщение об отсутствии. [web:2][web:4]

### Агент и AgentExecutor

Агент создаётся через `create_tool_calling_agent(llm, tools, prompt)`.  
Оркестрация работы — через `AgentExecutor`:
- принимает `input` от пользователя;
- запускает цикл: LLM → tool call → вызов функции → результат → LLM → ответ;
- возвращает финальный `output` пользователю. [web:2][web:7]

## Где происходит «принятие решений» агентом

- **На уровне LLM**  
  Модель решает:
  - нужно ли задавать уточняющие вопросы;
  - когда и какие инструменты вызывать;
  - какие аргументы передать инструменту;
  - как сформулировать ответ пользователю на основе результата.  
  Это основная «интеллектуальная» часть принятия решений. [web:2][web:6]

- **На уровне агента и AgentExecutor**  
  Реализация цикла:
  - интерпретирует tool call от LLM;
  - реально вызывает Python‑функцию;
  - возвращает результат в контекст;
  - повторяет цикл, пока LLM не решит, что ответ готов.  
  Здесь «техническая» часть решения: как и когда выполнять инструмент. [web:2][web:7]

- **На уровне разработчика (жёсткие правила)**  
  Разработчик задаёт рамки:
  - промпт явно требует сначала уточнять марку/модель/год;
  - список инструментов фиксирован (только поиск по каталогу);
  - цикл в `main()` задаёт интерактивный режим и завершение по команде.  
  LLM не может выйти за эти рамки. [web:2]
